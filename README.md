## Features

- Choose 1 item of each category for making the order
- After choosing the items for all the categories, the user can finish the order
- Order summary for user confirmation
- CSS classes dinamically inserted via JS to give a visual feedback to the user

## Requirements

 - Version control
   -  [x]  Use git versioning on a public github repository
   -  [x]  Make a commit after every feature added
- Layout
    - [x] Apply a mobile layout provided on figma (it doesn't need to have a desktop layout)
    - [x] The header and the "close order" button must be fixed on the page
    - [x]  Fill the categories with some options and different prices
    - [x]  Use the following fonts: [Righteous](https://fonts.google.com/specimen/Righteous) (app name and categories text) e [Roboto](https://fonts.google.com/specimen/Roboto) (everything else)
    - [x]  Apply shadows
    - [x]  Let the options scroll horizontally when needed
- Item selection
    - [x]  When clicking on an item, it must be selected
    - [x]  When clicking on another item at the same category, the first item must be unselected and the new item must be selected
    - [x]  On clicking an already selected item, there's no need to unselect it
- Finish order button
    - [x]  By default, the finish order button must start disabled. Nothing should happen when clicking on this button on that state.
    - [x]  When the user has already selected items from the 3 categories, the button must change it state and be clickable
- Send order
    - [x]  When confirming the order, the user must be redirected to a Whatsapp Web page with an already filled in message containing the order summary and addressed to the restaurant.
        
    - [x]  The message must follow the following format: 
        
        ```css
        Hello, I would like to make an order:
        - Dish: Yin Yang Chiken
        - Beverage: Cold Coca-Cola
        - Dessert: Pudding
        Total: R$ 27.70
        ```
